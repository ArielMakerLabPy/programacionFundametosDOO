    2. Identificando las clases en la aplicación

      3. Identificando objetos
      Escenario de caso de uso
      Subrayar los sustantivos dentro de éste ya que estos representan posiblemente los objetos
      Creamos una lista de nombres de los sustantivos comprobando si hay duplicados y deshacese de uno de ellos

    3. Definiendo las relaciones entre clases

    4. Identificando las responsabilidades de las clases

      5. Escenario de caso de uso:
      Identificar los comportamientos y cuales son las responsabilidades y para quienes que no siempre es obvio
      Distribuir las responsabilidades entre los objetos conceptuales (cajas creadas)
      Evita crear maestros objetos globales
      Deben estar bien distribuidas, tratando de repartir adecuandamente las acciones de los objetos
